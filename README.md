# Программный комплекс "Книжный магазин"

## Аннотация

Настоящий программный продукт представляет собой автоматизированную систему управления книжным фондом, реализованную в виде настольного приложения на платформе .NET 8.0. Проект разработан в соответствии с принципами объектно-ориентированного программирования и включает независимую библиотеку классов, графический интерфейс пользователя на базе Windows Forms и комплексный набор модульных тестов с полным покрытием кода.

## Теоретические основы

### Объектно-ориентированный подход

Разработанная система базируется на объектно-ориентированной парадигме программирования. Предметная область декомпозирована на три основные сущности: `Book` (книга), `BookShelf` (книжный шкаф) и `Shop` (магазин). Каждая сущность инкапсулирует собственное состояние и поведение, обеспечивая целостность данных и предотвращая несанкционированный доступ к внутренним структурам.

Архитектура приложения следует принципам разделения ответственности (Separation of Concerns), где бизнес-логика полностью изолирована от логики представления. Это достигается путем размещения классов предметной области в отдельной сборке `BookShopLibrary.dll`, которая может быть повторно использована в различных типах приложений (консольных, веб-приложениях, мобильных приложениях).

### Математическая модель

Основной вычислительной операцией в системе является поиск книги по заданному критерию (название или идентификатор). Поскольку книги распределены по шкафам, алгоритм поиска представляет собой двойной вложенный цикл. Временная сложность данного алгоритма описывается следующим выражением:

```
T(n) = Σ(i=1 to N) Σ(j=1 to Mᵢ) O(1) = O(N × M)
```

где:
- `T(n)` — временная сложность алгоритма поиска
- `N` — количество шкафов в магазине
- `Mᵢ` — количество книг в i-том шкафу
- `M` — максимальная вместимость одного шкафа

Таким образом, алгоритм имеет линейную сложность относительно общего числа книг в системе.

### Вероятность коллизий

При автоматической генерации книг возникает задача предотвращения дублирования названий. Вероятность коллизии (совпадения названий) при случайном выборе из базы данных описывается формулой:

```
P(A) = 1 - ∏(i=1 to n-1) (1 - i/K)
```

где:
- `P(A)` — вероятность возникновения коллизии
- `n` — текущее количество сгенерированных книг
- `K` — общее количество уникальных названий в базе данных

Для предотвращения коллизий реализован алгоритм проверки уникальности с автоматическим добавлением числового суффикса к повторяющимся названиям.

## Архитектура системы

Программный комплекс состоит из трех независимых проектов:

1. **BookShopLibrary** — библиотека классов, содержащая бизнес-логику
2. **BookShop** — графический интерфейс пользователя (Windows Forms)
3. **BookShop.Tests** — набор модульных тестов (xUnit)

Диаграмма зависимостей:

```
BookShop (UI) ──────> BookShopLibrary (Core Logic)
                              ↑
BookShop.Tests ───────────────┘
```

Такая архитектура обеспечивает:
- Повторное использование бизнес-логики
- Независимое тестирование компонентов
- Возможность замены пользовательского интерфейса без изменения ядра системы

## Установка и запуск

### Системные требования

- Операционная система: Windows 10/11 (x64)
- Платформа: .NET 8.0 Runtime или выше
- Оперативная память: минимум 512 МБ
- Свободное место на диске: 100 МБ

### Установка .NET SDK

1. Загрузите .NET 8.0 SDK с официального сайта Microsoft:
   ```
   https://dotnet.microsoft.com/download/dotnet/8.0
   ```

2. Запустите установщик и следуйте инструкциям мастера установки

3. Проверьте установку, выполнив команду в терминале:
   ```bash
   dotnet --version
   ```

### Клонирование репозитория

```bash
git clone https://github.com/IrishinT/BookShop.git
cd BookShop
```

### Сборка проекта

```bash
dotnet restore
dotnet build --configuration Release
```

### Запуск приложения

```bash
cd BookShop
dotnet run
```

Альтернативный способ — открыть файл решения `BookShop.sln` в Visual Studio 2022 и нажать F5.

### Запуск тестов

```bash
dotnet test
```

### Генерация отчета о покрытии кода

```bash
dotnet test --collect:"XPlat Code Coverage"
reportgenerator -reports:"BookShop.Tests\TestResults\**\coverage.cobertura.xml" -targetdir:"BookShop.Tests\TestResults\CoverageReport" -reporttypes:Html
```

Отчет будет доступен по пути:
```
BookShop.Tests\TestResults\CoverageReport\index.html
```

## Тестирование

### Методология

Для обеспечения корректности работы системы разработан комплексный набор модульных тестов (Unit Tests) с использованием фреймворка xUnit. Тестирование проводится по методологии AAA (Arrange-Act-Assert):

- **Arrange** (Подготовка) — инициализация тестовых данных
- **Act** (Действие) — выполнение тестируемого метода
- **Assert** (Проверка) — верификация результата

### Покрытие кода

Достигнуто полное покрытие кода:

- **Покрытие строк (Line Coverage):** 100%
- **Покрытие веток (Branch Coverage):** 100%

Всего разработано 48 тестовых сценариев, охватывающих:

- Корректность инициализации объектов
- Валидацию входных данных
- Обработку граничных случаев
- Проверку исключительных ситуаций
- Логику бизнес-процессов

### Примеры тестовых сценариев

#### Тест 1: Создание книги

```csharp
[Fact]
public void Constructor_ShouldCreateBookWithCorrectProperties()
{
    // Arrange & Act
    var book = new Book(1, "Тестовая книга", "Тестовый автор", "Фантастика", 300, 500m);

    // Assert
    Assert.Equal(1, book.Id);
    Assert.Equal("Тестовая книга", book.Title);
    Assert.Equal("Тестовый автор", book.Author);
    Assert.Equal("Фантастика", book.Genre);
    Assert.Equal(300, book.Pages);
    Assert.Equal(500m, book.Price);
}
```

#### Тест 2: Проверка переполнения шкафа

```csharp
[Fact]
public void AddBook_ShouldReturnFalse_WhenShelfIsFull()
{
    // Arrange
    var shelf = new BookShelf(1, "Фантастика", 2);
    var book1 = new Book(1, "Книга 1", "Автор", "Фантастика", 200, 500m);
    var book2 = new Book(2, "Книга 2", "Автор", "Фантастика", 200, 500m);
    var book3 = new Book(3, "Книга 3", "Автор", "Фантастика", 200, 500m);

    // Act
    shelf.AddBook(book1);
    shelf.AddBook(book2);
    var result = shelf.AddBook(book3);

    // Assert
    Assert.False(result);
    Assert.Equal(2, shelf.CurrentCount);
}
```

#### Тест 3: Проверка несоответствия жанров

```csharp
[Fact]
public void AddBook_ShouldThrowException_WhenGenreMismatch()
{
    // Arrange
    var shelf = new BookShelf(1, "Фантастика", 10);
    var book = new Book(1, "Книга", "Автор", "Детектив", 200, 500m);

    // Act & Assert
    Assert.Throws<InvalidOperationException>(() => shelf.AddBook(book));
}
```
